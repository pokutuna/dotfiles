現在のセッションの引き継ぎドキュメントを作成し、現在の作業ディレクトリに保存せよ。

## ファイル名

```
HANDOVER-$(date +%y%m%dT%H%M).md
```

## ドキュメントの内容

次のセッションで即座に作業を再開できるよう、以下を含めること。簡潔かつ完全に — 重複作業や同じミスを防げる情報は積極的に含める。

### 1. Task Overview
- ユーザーの要求と成功基準
- 指定された制約や clarification

### 2. Current State
- 完了済みの作業
- 作成・変更・分析したファイル(パス付き)
- 生成した成果物

### 3. Important Discoveries
- 判明した技術的制約や要件
- 下した判断とその理由
- 発生したエラーと解決方法
- 試したが上手くいかなかったアプローチとその理由

### 4. Next Steps
- タスク完了に必要な具体的アクション
- ブロッカーや未解決の疑問点
- 複数ステップが残る場合の優先順位

### 5. Context to Preserve
- ユーザーの好みやスタイル要件
- 自明でないドメイン固有の情報
- ユーザーへの約束事
